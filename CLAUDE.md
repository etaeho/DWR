## 파일 구조

- `루틴.md` — 일일 식단 + 운동 루틴 | **이력 보존 필수**
- `신체상태.md` — 날짜별 신체 정보 누적 (최신이 상단)

## 이력 보존 규칙 (운동목표.md / 루틴.md)
- 내용 변경 시 기존 내용을 삭제하지 않고 `## [이력]` 섹션으로 날짜��� 함께 이동
- 새 내용은 항상 `## [현재] YYYY-MM-DD ~` 형식으로 상단에 추가
- 예: 목표 수정 → 기존 [현재] 블록을 [이력]으로 내리고, 새 목표를 [현재]로 추가

## 핵심 동작 규칙
1. 식단/운동/신체 정보 입력 → 날짜 폴더 자동 생성 후 해당 파일에 기록
2. 식단 분석은 항상 `운동목표.md`의 [현재] 목표 수치 기준
3. 날짜 미언급 시 오늘 날짜(YYYY-MM-DD) 사용
4. 같은 날 추가 입력 시 기존 파일에 이어서 추가 (덮어쓰기 금지)
5. `신체상태.md`는 최신 날짜가 파일 상단
6. 이미지 인식 불가 항목은 `추정` 표기 후 기록
7. 식단/운동 기록 후 `## 📊 대시보드` 섹션을 파일 하단에 자동 출력
   - 영양 누계 / 근손실 위험 / 혈당 스파이크 세 표를 하나의 섹션 안에 순서대로 배치
   - 식사 추가 시마다 해당 섹션을 최신 수치로 덮어씀 (섹션 전체 교체)
   - **영양 누계**: 칼로리·단백질·탄수·지방 섭취 vs 목표 vs 달성률
   - **근손실 위험**: 일일 목표 달성률이 아닌 "지금 이 순간 근육이 분해되고 있는가" 현재 상태 판정
     - 판정 항목 3가지:
       1. 공복 시간 — 마지막 식사로부터 경과: 4h 이내 🟢 / 4~6h 🟡 / 6h+ 🔴
       2. 운동 후 단백질 — 당일 운동이 있을 경우 운동 후 단백질 섭취: 20g+ 🟢 / 미흡 🟡 / 미섭취 🔴 / 운동 없으면 표시 생략
       3. 저항운동 — 최근 웨이트 시점: 오늘 🟢 / 어제 🟡 / 2일+ 공백 🔴
     - 종합: 🔴 하나라도 있으면 🔴 위험 / 🟡만 있으면 🟡 주의 / 모두 🟢면 🟢 안전
   - **혈당 스파이크**: 식사별 GL | 예상 최고혈당(공복90 + GL×2.5) | 판정
     - 140 이하 🟢 정상 / 140~160 🟡 주의 / 160 초과 🔴 스파이크
     - 반드시 `(추정) ⚠️ 실측값 아님` 표기
