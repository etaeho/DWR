## 이력 보존 규칙 (운동목표.md / 루틴.md)
- 내용 변경 시 기존 내용을 삭제하지 않고 `## [이력]` 섹션으로 날짜��� 함께 이동
- 새 내용은 항상 `## [현재] YYYY-MM-DD ~` 형식으로 상단에 추가
- 예: 목표 수정 → 기존 [현재] 블록을 [이력]으로 내리고, 새 목표를 [현재]로 추가

## 핵심 동작 규칙
1. 식단/운동/신체 정보 입력 → 날짜 폴더 자동 생성 후 해당 파일에 기록
2. 식단 분석은 항상 `운동목표.md`의 [현재] 목표 수치 기준
3. 날짜 미언급 시 오늘 날짜(YYYY-MM-DD) 사용
4. 같은 날 추가 입력 시 기존 파일에 이어서 추가 (덮어쓰기 금지)
5. `신체상태.md`는 최신 날짜가 파일 상단
6. 이미지 인식 불가 항목은 `추정` 표기 후 기록
7. 식단/운동 기록 후 대시보드 자동 출력 — 아래 두 섹션 포함
   - **근손실 위험 지표**: 단백질 충족률 + 칼로리 적자 기준으로 🟢/🟡/🔴 판정
     - 🟢 안전: 단백질 80%+ & 칼로리(섭취/TDEE) 70%+
     - 🟡 주의: 둘 중 하나 미달
     - 🔴 위험: 단백질 60% 미만 or 칼로리 60% 미만
     - 표시 형식: `단백질 충족률 | 칼로리 적자 | 종합 판정` 한 줄 표
8. 식단 기록 시 혈당 스파이크 체크 섹션 자동 추가 (간소화)
   - 식사별 GL 합계 계산: 탄수화물(g) × (GI / 100)
   - 예상 최고혈당 = 공복 90 + GL × 2.5 mg/dL
   - 판정: 140 이하 🟢 정상 / 140~160 🟡 주의 / 160 초과 🔴 스파이크
   - 표시 형식: 식사별 한 줄 요약 표 (GL | 예상 최고혈당 | 판정)
   - 반드시 `(추정) ⚠️ 실측값 아님` 표기
